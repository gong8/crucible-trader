CRUCIBLE TRADER
Phase 1 Specification — Statistical Validation and Live Data Integration (v1.0)

────────────────────────────────────────────
	1.	PHASE 1 MISSION
────────────────────────────────────────────
Phase 1 transforms Crucible Trader from a research backtester into a rigorous statistical validation platform with live market data integration.

Core Goals:
	1.	Statistical Robustness – Detect overfitting through permutation tests, walk-forward analysis, and bootstrap validation.
	2.	Live Data Integration – Replace CSV reliance with Tiingo API as the default data source.
	3.	Advanced Analytics – Monte Carlo simulation, parameter optimization, and regime detection.
	4.	Production Metrics – Integrate C++ quant service for high-performance calculations.
	5.	Enhanced Reporting – Statistical p-values, out-of-sample performance, and confidence intervals.

Mantra:
"Backtest results mean nothing without statistical validation. Phase 1 separates signal from noise."

	5.	STATISTICAL VALIDATION METHODS
────────────────────────────────────────────

5.1 Permutation Testing
Purpose: Detect if backtest results are due to chance.
Method:
	1.	Run original backtest → get metric M (e.g., Sharpe = 1.5)
	2.	Shuffle trade entry/exit signals 1,000 times
	3.	Re-run backtest with shuffled signals
	4.	Calculate p-value = (# shuffled results ≥ M) / 1,000
	5.	If p < 0.05, results are statistically significant

Implementation:
POST /api/stats/permutation
{
  "runId": "original-run-id",
  "metric": "sharpe",
  "iterations": 1000,
  "seed": 42
}

Response:
{
  "testId": "perm-123",
  "originalMetric": 1.5,
  "pValue": 0.023,
  "nullDistribution": [...],
  "conclusion": "significant"
}

5.2 Walk-Forward Analysis
Purpose: Validate strategy on unseen data to detect overfitting.
Method:
	1.	Split data into windows (e.g., 12 months IS, 3 months OOS)
	2.	Optimize parameters on IS window
	3.	Test on OOS window
	4.	Roll forward and repeat
	5.	Aggregate OOS performance

Implementation:
POST /api/stats/walkforward
{
  "strategyName": "sma_crossover",
  "symbol": "AAPL",
  "timeframe": "1d",
  "start": "2020-01-01",
  "end": "2024-12-31",
  "inSampleMonths": 12,
  "outSampleMonths": 3,
  "paramGrid": {
    "fastLength": [5, 10, 15],
    "slowLength": [20, 30, 40]
  },
  "objective": "sharpe"
}

Response:
{
  "walkforwardId": "wf-456",
  "windows": [
    {
      "window": 1,
      "inSample": { "start": "2020-01-01", "end": "2020-12-31" },
      "outSample": { "start": "2021-01-01", "end": "2021-03-31" },
      "bestParams": { "fastLength": 10, "slowLength": 30 },
      "inSampleSharpe": 1.8,
      "outSampleSharpe": 0.9
    },
    ...
  ],
  "aggregateOOS": {
    "sharpe": 1.1,
    "maxDD": -0.15,
    "winrate": 0.58
  }
}

5.3 Bootstrap Resampling
Purpose: Estimate confidence intervals for metrics.
Method:
	1.	Resample returns with replacement 1,000 times
	2.	Calculate metric on each bootstrap sample
	3.	Build distribution → 95% confidence interval

Implementation:
POST /api/stats/bootstrap
{
  "runId": "original-run-id",
  "metric": "sharpe",
  "iterations": 1000,
  "confidenceLevel": 0.95
}

Response:
{
  "metric": "sharpe",
  "point": 1.5,
  "ci95Lower": 1.2,
  "ci95Upper": 1.8,
  "standardError": 0.15
}

5.4 Monte Carlo Simulation
Purpose: Stress-test strategy under different market scenarios.
Method:
	1.	Use C++ quant service for performance
	2.	Generate synthetic price paths (GBM, Heston, etc.)
	3.	Run strategy on each path
	4.	Aggregate results → probability distribution

Implementation:
POST /api/stats/montecarlo
{
  "runId": "original-run-id",
  "model": "gbm", // 'gbm', 'heston', 'jump_diffusion'
  "paths": 10000,
  "horizonDays": 252,
  "seed": 42
}

Response:
{
  "mcId": "mc-789",
  "expectedReturn": 0.12,
  "stdDev": 0.18,
  "var95": -0.22,
  "cvar95": -0.31,
  "percentiles": {
    "p5": -0.15,
    "p50": 0.11,
    "p95": 0.42
  }
}

All fetched data goes through validation:
	•	No missing dates (within market hours)
	•	No negative prices
	•	No extreme outliers (> 10 std dev moves)
	•	Volume > 0 for traded bars
	•	OHLC consistency (O/H/L/C relationships)

	1.	Attempt Polygon.io
	2.	Fall back to cached data
	3.	Fail with clear error if all sources exhausted

────────────────────────────────────────────
	8.	ENHANCED METRICS
────────────────────────────────────────────

New metrics in packages/metrics/src/advanced.ts:

Value at Risk (VaR)
Maximum expected loss at 95% confidence over N days.
var95 = percentile(returns, 0.05)

Conditional Value at Risk (CVaR)
Average loss beyond VaR threshold.
cvar95 = mean(returns where returns < var95)

Omega Ratio
Probability-weighted ratio of gains to losses.
omega = sum(returns > threshold) / sum(returns < threshold)

Calmar Ratio
Return / Max Drawdown (annualized).
calmar = cagr / abs(max_dd)

Recovery Factor
Total return / Max Drawdown.
recovery = total_pnl / abs(max_dd_dollars)

Tail Ratio
95th percentile return / 5th percentile return.
tail = percentile(returns, 0.95) / abs(percentile(returns, 0.05))

Drawdown Analysis
	•	Underwater curve (equity below peak)
	•	Drawdown duration (days underwater)
	•	Recovery time (days to new high)
	•	Max consecutive losing days

	9.	REGIME DETECTION
────────────────────────────────────────────

Purpose: Identify market conditions and adapt strategies.

Regime Types:
	1.	Trend: Bull, Bear, Sideways
	2.	Volatility: Low, Medium, High
	3.	Combined: Bull+LowVol, Bear+HighVol, etc.

Detection Methods:
	•	Moving Average Slopes (trend)
	•	ATR / Historical Volatility (volatility)
	•	Hidden Markov Models (HMM)
	•	Change Point Detection

Implementation:
POST /api/regimes/detect
{
  "symbol": "AAPL",
  "timeframe": "1d",
  "start": "2020-01-01",
  "end": "2024-12-31",
  "method": "hmm", // 'ma', 'hmm', 'volatility'
  "states": 3 // bull, bear, sideways
}

Response:
{
  "regimes": [
    {
      "type": "bull",
      "start": "2020-01-01",
      "end": "2020-02-15",
      "confidence": 0.89
    },
    {
      "type": "bear",
      "start": "2020-02-16",
      "end": "2020-03-31",
      "confidence": 0.94
    },
    ...
  ],
  "strategyPerformance": {
    "bull": { "sharpe": 2.1, "trades": 15 },
    "bear": { "sharpe": -0.5, "trades": 8 },
