## Purpose

Capture the minimal Phase 0 UI contract: Next.js App Router pages for runs, new run creation, datasets, risk management, and reports.

## Inputs

- `NEXT_PUBLIC_API_BASE_URL` env var pointing to the running Fastify API.
- Strategy presets defined in `apps/web/src/app/new-run/page.tsx`.
- Dataset metadata returned from the API.

## Outputs

- `/runs` list showing name, status, key metrics.
- `/new-run` form with auto data source, strategy dropdown, JSON preview, and error toasts.
- `/datasets`, `/risk`, `/reports` stubbed pages for dataset registration, risk editing, and future reporting UI.

## Invariants

- UI never stores secrets; everything is client-side fetch to API.
- Inputs default to valid values (auto data source, SMA preset, sample symbol/timeframe).
- Status indicators rely solely on API responses (no client guesses).

## Example

User selects “auto (prefer cached)” data source, chooses `sma_crossover`, hits “launch run” → UI renders run ID and surfaces any dataset errors coming from Fastify.

## Test Checklist

- `pnpm --filter @crucible-trader/web build` passes.
- Manual smoke: `pnpm --filter @crucible-trader/web dev` shows forms working and remote requests hitting the API successfully.
